## 设计理念

### 核心价值观
**工程实用性 × 现代审美**

- **简洁高效**：减少操作步骤，快速完成工作
- **对比鲜明**：确保在强光和弱光环境都能清晰识别
- **信息层级**：重要信息优先展示
- **无障碍友好**：大按钮、高对比度、清晰标签

### 设计原则
1. **工程优先**：所有设计决策以实际工作流程为基础
2. **信息准确**：关键数据始终清晰可见，不让用户猜测
3. **反馈即时**：操作后立即给出视觉反馈
4. **一致性**：所有页面遵循统一的设计语言

## 色彩系统

### 主色板

| 色彩名称 | 十六进制 | RGB | 用途 | 说明 |
|---------|--------|-----|------|------|
| 品牌蓝 | `#2196F3` | (33, 150, 243) | 按钮、链接、突出显示 | 主要交互元素 |
| 成功绿 | `#4CAF50` | (76, 175, 80) | 正常状态、确认 | 表示操作成功或正常状态 |
| 警告橙 | `#FF9800` | (255, 152, 0) | 注意状态、同步中 | 需要关注的状态 |
| 错误红 | `#FF6B6B` | (255, 107, 107) | 异常、警告、删除 | 表示错误或需要立即处理 |

### 辅助色板

| 色彩名称 | 十六进制 | 用途 |
|---------|--------|------|
| 纯白 | `#FFFFFF` | 主背景、卡片背景 |
| 浅灰 | `#F5F5F5` | 次级背景、卡片内容背景 |
| 中灰 | `#E0E0E0` | 分隔线、边框 |
| 深灰 | `#808080` | 次要文字、禁用状态 |
| 黑色 | `#000000` | 主要文字、标题 |
| AR黑 | `#1A1A1A` | AR页面背景 |
| AR深黑 | `#2A2A2A` | AR页面卡片背景 |

### 透明度使用
```
完全不透明: alpha = 1.0 (100%)
主要内容: alpha = 0.87  (87%)
次要内容: alpha = 0.60  (60%)
禁用状态: alpha = 0.38  (38%)
极度淡化: alpha = 0.12  (12%)
```

### 状态色映射

```
正常/成功  ──→ 🟢 #4CAF50 (绿色)
注意/警告  ──→ 🟡 #FF9800 (橙色)
异常/错误  ──→ 🔴 #FF6B6B (红色)
中立/默认  ──→ #2196F3   (蓝色)
```

## 排版系统

### 字体设置
- **默认字体**：系统默认（中文：黑体 / 英文：Roboto）
- **特殊场景**：等宽字体用于坐标/数据显示

### 字号与权重

| 用途 | 字号 | 粗细 | 行高 | 示例 |
|------|------|------|------|------|
| 页面标题 | 20sp | Bold (700) | 28sp | "测量" |
| 段落标题 | 16sp | SemiBold (600) | 24sp | "GPS 定位信息" |
| 卡片标题 | 15sp | SemiBold (600) | 22sp | "北京中关村项目" |
| 正文 | 14sp | Regular (400) | 20sp | "X: 1024.5m" |
| 标签/辅助 | 12sp | Regular (400) | 18sp | "纬度", "精度" |
| 小标签 | 11sp | Regular (400) | 16sp | "数据点", "大小" |
| 极小文字 | 10sp | Regular (400) | 14sp | Badge标签 |

### 字色应用

| 层级 | 颜色 | 用途 |
|------|------|------|
| 一级 | `#000000` (alpha=1.0) | 主要文字、标题 |
| 二级 | `#333333` (alpha=0.87) | 正文、重要信息 |
| 三级 | `#808080` (alpha=0.6) | 辅助文字、时间 |
| 四级 | `#CCCCCC` (alpha=0.38) | 禁用文字、提示 |

## 间距与布局

### 基础间距单位

```
1 单位 = 4dp

常用间距:
- 极小: 4dp   (1 unit)
- 小:  8dp   (2 units)
- 中:  12-16dp (3-4 units)
- 大:  24dp   (6 units)
- 极大: 32dp   (8 units)
```

### 应用间距

| 场景 | 间距 | 说明 |
|------|------|------|
| 屏幕边距 | 16dp | 页面周围的基本留白 |
| 卡片内部 | 16dp | 卡片padding |
| 卡片之间 | 12dp | 卡片垂直间隔 |
| 行内元素 | 12dp | 同行元素之间 |
| 组件内部 | 12dp | 标题与内容之间 |
| 按钮间距 | 12dp | 并排按钮之间 |

### 布局网格
```
页面总宽度 = 屏幕宽度
有效内容宽度 = 屏幕宽度 - 32dp (左右各16dp)
最小可点击区域 = 48×48dp
按钮最小高度 = 48dp (推荐: 56dp)
```

```

颜色: 品牌蓝 (#2196F3)
边框: 1dp
高度: 48dp
圆角: 8dp
字大: 14sp, Regular
```

#### 危险操作按钮（Danger Button）

## 交互规范

### 触摸反馈

| 元素类型 | 按下效果 | 持续时间 |
|---------|---------|---------|
| 按钮 | 背景加深 + 阴影增加 | 200ms |
| 卡片 | 阴影增加 + 轻微缩放 | 150ms |
| 开关 | 滑块响应 | 300ms |
| 文本框 | 边框变色 + 光标出现 | 即时 |

### 加载状态

```
显示方式: 
  1. 圆形进度条 (定量加载)
  2. 线条进度条 (确定进度)
  3. 脉冲动画 (不确定加载)

按钮加载:
  文字变为 "加载中.../" 
  禁用按钮交互
  显示加载动画
```

### 反馈提示

#### Toast通知（临时）
```
显示时长: 2-3秒自动消失
位置: 底部中央
内容: 简短反馈信息
示例: "数据已保存"
```

#### Dialog对话框（重要）
```
用途: 确认删除、重要警告
显示: 模态框, 半透明背景
按钮: 确定 + 取消
```

#### Snackbar
```
用途: 操作反馈、撤销提示
位置: 底部
自动消失: 3-4秒
可长按查看: 完整信息
```

### 页面转换

```
页面切换: 无动作 (响应式架构)
导航更新: 淡入淡出 (200ms)
卡片展开: 向上滑入 (300ms)
返回动作: 向下滑出 (200ms)
```

## 无障碍设计

### 对比度要求
```
WCAG AA 标准:
- 大文字: 最少 3:1
- 普通文字: 最少 4.5:1
- UI组件: 最少 3:1
```

### 触摸目标
```
最小可点击区域: 48×48dp
按钮最小高度: 48-56dp
按钮最小宽度: 48dp
按钮间距: ≥12dp
```

### 文字标注
```
所有输入框都有文字标签
所有图标都有描述文字
颜色不是唯一信息载体
```

## 验证清单

在实现UI时，确保以下各项都满足requirements：

- [ ] 所有文字对比度 ≥ 4.5:1
- [ ] 按钮高度 ≥ 48dp
- [ ] 间距遵循 4dp 网格
- [ ] 圆角使用标准值 (4/8/12/16dp)
- [ ] 所有交互都有视觉反馈
- [ ] 加载状态明确显示
- [ ] 错误信息清晰易懂
- [ ] 图标有 contentDescription
- [ ] 页面在各尺寸设备都可用
- [ ] 正文字大最小 12sp
